# LLM Behavior Lab

> 面向大语言模型行为研究的实验设计、运行与初步分析平台。

LLM Behavior Lab 的目标不是再做一个通用聊天机器人评测页，而是把一个模糊的研究兴趣转化为可审查、可运行、可复现的 LLM 行为实验。它面向希望快速开展预实验的社会科学、心理学、HCI、AI safety、AI evaluation 与计算社会科学研究者，尤其适合那些已经有研究问题，但还没有把问题拆成实验条件、测量指标和分析方案的早期阶段。

平台的核心假设是：随着大语言模型越来越频繁地被作为实验对象、模拟被试、交互式代理或社会技术系统的一部分来研究，研究者需要的不只是“写几个 prompt 看输出”，而是一套能把行为科学实验设计、心理构念操作化、benchmark 测量、模型 API 调度和统计分析连接起来的工作流基础设施。

## 产品定位

LLM Behavior Lab 是一个 experiment copilot 与 execution workbench 的结合体。

研究者可以用自然语言输入一个问题，例如“喝咖啡是否会影响大语言模型在任务中的表现”“合作情境是否会让模型更公平”“模型在收益和损失框架下是否表现出不同风险偏好”。平台会把这个问题逐步转译为实验协议：

- 从研究意图中识别候选心理、社会或认知构念。
- 判断哪些构念应被设计为实验操纵、自变量、因变量、控制变量或协变量。
- 建议组间、组内或混合设计，并生成条件矩阵。
- 推荐适合的测量形式，包括 Likert 评分、forced choice、自由文本编码、正确率、反应时、拒答率、置信度、解释质量、benchmark 分数等。
- 生成不同条件下的 prompt 草案、输出 schema、模型运行计划和数据记录字段。
- 调用模型 runner 批量运行，并保留原始响应、解析结果、模型参数、时间戳、延迟、token/成本和 prompt 版本。
- 输出可快速审阅的描述统计、条件比较、可视化图表、限制说明和导出材料。

换句话说，它试图把“大模型行为实验”从一次性的手工 prompt 调试，推进为协议优先、数据可追溯、可迭代复现实验的产品化流程。

## 为什么需要

传统人类行为实验平台已经形成了相对成熟的研究链路：实验设计、条件分配、被试招募、行为记录、数据导出与统计分析。但 LLM 行为实验仍常常停留在临时脚本和临时 prompt 的层面，容易出现几个问题：

- 研究问题没有被明确操作化，导致自变量、因变量和控制条件混在一起。
- prompt 差异同时改变了任务、语气、信息量和诱导方式，难以判断效应来源。
- 模型、temperature、重复采样次数、解析规则和失败样本没有被系统记录。
- 输出分析依赖手工阅读，难以快速比较条件、模型和重复试次。
- 研究者需要同时懂实验设计、prompt engineering、API 调用、数据清洗和统计分析，上手成本过高。

LLM Behavior Lab 希望承担其中的中间层：让研究者仍然保留理论判断和实验确认权，但把低层的拆解、协议化、运行和初步分析交给系统完成。

## 核心工作流

### 1. 需求输入

用户以文字、语音转录或上传材料的方式描述研究问题。输入可以是不完整的想法、已有实验方案、论文摘要、聊天记录或 benchmark 线索。系统首先生成一份可编辑的研究目标摘要，帮助用户确认“平台是否理解了真正想问的问题”。

### 2. 构念与变量拆解

系统将自然语言问题拆解为更可检验的研究结构：

- 目标构念：例如公平偏好、风险偏好、合作倾向、社会顺从、道德判断、信息寻求、信任、反应一致性。
- 操纵变量：哪些内容可以通过 prompt、情境、任务材料或反馈规则来改变。
- 测量变量：哪些行为输出可以作为因变量或中介性指标。
- 控制要素：模型、任务难度、文本长度、输出格式、顺序、随机化和重复采样。
- 不可直接实现的部分：例如“模型是否真的喝咖啡”会被转译为“喝咖啡相关语境或身份提示是否影响下游任务表现”。

这一层的重点不是替用户决定理论，而是把隐含的理论假设暴露出来，让用户可以修改、删除或确认。

### 3. 实验设计建议

平台会基于问题类型建议可运行的设计形式：

- 组间设计：不同模型调用或不同样本只接受一个条件。
- 组内设计：同一模型在多个条件下重复完成任务，适合控制模型差异。
- 混合设计：条件操纵与模型类型、任务类型或 benchmark 类别交叉。
- 条件矩阵：明确实验组、对照组、随机化策略、重复试次和最小 preview run。
- 质量控制：提示词长度平衡、输出字段约束、条件标签隐藏、解析失败处理和异常样本标记。

系统会优先给出能快速跑通的预实验方案，而不是一开始就把用户推入复杂的大规模研究设计。

### 4. 测量与 benchmark 推荐

平台会根据研究问题建议一组候选测量方式。不同实验可以选择不同粒度：

- 主观评分：Likert 量表、置信度、偏好强度、接受度、风险倾向。
- 选择行为：A/B 选择、多选、接受/拒绝、合作/背叛、是否遵循规则。
- 任务表现：正确率、得分、benchmark accuracy、pass rate、rubric score。
- 过程指标：反应时、延迟、token 长度、解释性文本比例、JSON 合规率、拒答率。
- 自由文本指标：理由编码、主题分类、道德框架、策略类型、语气与立场。
- 经典材料线索：相关心理量表、行为任务范式、公开 benchmark 或领域任务集合。

用户可以把平台建议视为“测量菜单”：先选择一个低成本主指标，再逐步补充更细的机制指标。

### 5. 协议与 prompt 编译

确认设计后，系统生成机器可运行的实验协议，包括：

- 研究问题与假设。
- 条件定义与条件矩阵。
- prompt 模板和每个条件的变量替换。
- 输出 schema 与解析规则。
- 模型列表、API 模式、重复试次和预算上限。
- preview run 与正式 run 的执行计划。
- 需要导出的原始数据和派生指标。

协议优先的好处是，prompt、模型和分析都服务于同一个可审查对象，而不是散落在不同脚本和聊天记录中。

### 6. 运行与数据记录

runner 按照 `condition x model x repetition` 执行任务。每条响应都会保留原始输出和结构化解析结果，并记录运行时元数据，例如模型名、参数、时间戳、延迟、token 近似、错误信息、解析 fallback 和成本估计。这样可以支持后续复查、排错和跨模型比较。

### 7. 初步分析与导出

平台提供面向预实验的快速分析：

- 条件均值、比例、分布和置信区间。
- 模型间与条件间的初步比较。
- 自由文本编码摘要与异常样本示例。
- 运行质量指标，如失败率、拒答率、解析率和延迟。
- 可导出的 CSV、JSON、实验协议和报告草稿。

这些结果不替代正式统计建模，但可以帮助研究者判断：这个问题是否值得扩大样本、是否需要换测量方式、是否需要引入人类被试基线。

## 当前版本

当前仓库包含一个可本地运行的 v1 原型，重点验证“从自然语言问题到可运行实验方案”的闭环：

- 研究需求输入与材料上传入口。
- 细节确认页，用于编辑研究目标摘要、假设问题和主要因变量测量形式。
- 实验协议页，用于审查条件矩阵和协议 JSON。
- 提示词检查页，用于查看不同条件的 prompt 与质量检查项。
- 模型与预算页，用于配置模型、API 模式、重复试次和成本上限。
- preview run、正式 run、结果报告与导出页面。
- 默认 simulator，可在没有外部 API 的情况下完整跑通流程。
- 可选 OpenAI-compatible chat completions 接口，用于接入真实模型。

## 适用场景

- 把经典心理学或行为经济学实验快速改写为 LLM 行为实验。
- 在正式人类实验前，先用模型 API 做机制预检和材料打磨。
- 比较不同模型在同一实验条件下的行为差异。
- 测试 prompt 条件、身份提示、情境 framing、任务材料变化是否影响输出。
- 为课程、工作坊或研究小组提供可展示的实验设计与运行流程。
- 为后续论文或项目申请生成更规范的预实验记录。

## 不是什么

LLM Behavior Lab 不试图替代完整的统计建模、伦理审查或人类被试研究，也不把模型输出直接等同于人类心理。它更适合作为一个协议化的研究前台：帮助研究者把想法变成可以讨论、运行、复查和迭代的实验对象。

它也不是 leaderboard 式的通用模型评分平台。平台关注的是“在某个理论问题下，如何构造条件、测量行为、记录过程并解释差异”，而不是单一总分排名。

## 本地运行

Windows 用户可以直接运行：

```cmd
run-server.cmd
```

或使用 Node：

```bash
node app/server.mjs
```

默认访问地址：

```text
http://127.0.0.1:8780/
```

如需接入真实模型，可在 `app/.env.local` 中配置 OpenAI-compatible 接口。不要把 API key 提交到仓库。

```env
COPILOT_LLM_ENDPOINT=https://example.com/v1/chat/completions
COPILOT_LLM_MODEL=your-model-name
COPILOT_LLM_API_KEY=your-api-key
```

## 路线图

- 更稳定的问题拆解 prompt 与多智能体审阅流程。
- 接入真实文献检索，把相近研究、经典量表和 benchmark 证据展示给用户。
- 支持更多实验类型，例如 repeated games、moral dilemmas、preference elicitation、social reasoning、instruction following 和 domain benchmarks。
- 引入更严格的统计模块，包括 bootstrap、混合效应模型、贝叶斯比较和多重比较提示。
- 支持团队工作空间、版本管理、评论与协作审查。
- 支持把 LLM 实验协议转换为人类被试兼容版本，作为后续 beta 功能。

## 一句话总结

LLM Behavior Lab 希望成为大语言模型行为实验的“从想法到预实验”的工作台：让研究者用自然语言开始，用实验协议约束，用模型运行验证，用数据和图表决定下一步。
