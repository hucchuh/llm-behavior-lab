# A-Class Issue / IV / DV / Operationalization Landscape

This is a collaborator-facing landscape for A-class literature: papers that use LLMs in substantive behavioral experiments, tasks, games, surveys, simulations, or interaction protocols.

It answers two linked questions:

1. What behavioral issues have LLMs been used to study, and in each paper what task, IV, IV condition, DV, DV measurement, and human baseline appear?
2. How did the authors operationalize those variables through constructs, contrasts, task materials, prompts, scenarios, game structures, model/agent conditions, or environments?

Boundary and evidence rule:

- Included: A-class substantive behavior experiments and A-B bridge papers only insofar as they contain a substantive LLM behavior experiment.
- Excluded here: method-validity evaluation as a separate topic. Persona validity, prompt fragility, construct-validity critique, and human-baseline problems belong in B-class synthesis.
- Source base: current `include` corpus, with role cleanup in `../survey_role_matrix.md`.
- Grain: issue family -> paper -> extracted note fields. Each row is grounded in an existing full-read note; candidate or maybe papers are not used as evidence.
- Papers can appear under multiple families when their note codes multiple substantive issues.

Current coverage:

- Current `include` papers: 43.
- A-compatible papers: 43.
- Primary A evidence: 31.
- A-compatible operationalization evidence: 3.
- A-B bridge papers: 9.
- First-layer A-class issue families: 17.
- Paper-family rows in each layer: 77.

# How To Read This Landscape

- Layer 1 is for issue coverage: what the paper studies, what it controls or manipulates, and what it measures.
- Layer 2 is for operationalization: how the paper turns constructs and IVs into concrete conditions, prompts, tasks, scenarios, games, model comparisons, or agent environments.
- Each section is an issue family.
- Each row is one paper within that family, not a merged family-level summary.
- `Specific issue / question` records what behavioral problem that paper uses LLMs to study.
- `Task / paradigm / scale` records the concrete behavioral task, game, survey, benchmark, vignette set, or simulation environment when specified in the note.
- `Controlled IV / manipulation` records the authored contrast: treatment, task condition, role, scenario, model/agent condition, persona/context, or environment condition.
- `IV levels / conditions` records the concrete levels or variants when the note specifies them.
- `DV construct / outcome` records the behavior or judgment observed.
- `DV measurement / metric` records how the behavior or judgment was measured.
- `Human baseline / comparison` records whether the paper compares to human data, human ratings, human experimental findings, or a normative/reference baseline.
- `Construct / target`, `IV operationalization`, `Condition contrast`, and `Task / material realization` are extracted from the Layer 2 fields of the corresponding full-read note.
- `Author caveat relevant to operationalization` reports source-level caveats when the note records them; it is not a new validity judgment by this survey.
- Some papers appear in more than one issue family because they study multiple behavioral outcomes.
- For role cleanup, see `../survey_role_matrix.md`; for compact single-row-per-paper detail, see `a_class_article_iv_dv_map.md`.
